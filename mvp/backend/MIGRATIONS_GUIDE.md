## 🎯 Introduction

### Qu'est-ce qu'Alembic ?

Alembic est un outil de **gestion de migrations de base de données** pour SQLAlchemy/SQLModel qui permet de :

- ✅ **Versionner** les changements de schema
- ✅ **Appliquer** les migrations de manière contrôlée
- ✅ **Rollback** en cas d'erreur
- ✅ **Générer** automatiquement les migrations à partir des models
- ✅ **Garder** un historique des modifications

### Pourquoi utiliser Alembic ?

#### ❌ Sans Alembic
```
Modification model → Supprimer la DB → Recréer → Perdre les données
```

#### ✅ Avec Alembic
```
Modification model → Générer migration → Appliquer → Données préservées
```

## ⚙️ Configuration

### Structure des Fichiers

```
backend/
├── alembic/
│   ├── versions/           # Fichiers de migration
│   │   └── XXXXXX_initial.py
│   ├── env.py              # Configuration environnement
│   ├── script.py.mako      # Template migrations
│   └── README
├── alembic.ini             # Configuration principale
└── scripts/
    ├── db-migrate.sh       # Script Linux/Mac
    └── db-migrate.bat      # Script Windows
```

### Configuration Async

Notre projet utilise **AsyncPG** pour PostgreSQL async. La configuration dans `alembic/env.py` est adaptée pour supporter l'async.

**Points clés** :
- `async_engine_from_config` au lieu de `engine_from_config`
- `asyncio.run()` pour exécuter les migrations
- Import de tous les models pour l'autogenerate

## 🚀 Commandes Essentielles

### 1. Générer une Migration

```bash
# Avec script (recommandé)
scripts\db-migrate.bat generate "description des changements"

# Ou directement
alembic revision --autogenerate -m "description des changements"
```

**Exemple** :
```bash
alembic revision --autogenerate -m "add full_name to user table"
```

### 2. Appliquer les Migrations

```bash
# Appliquer toutes les migrations en attente
scripts\db-migrate.bat upgrade

# Ou directement
alembic upgrade head
```

### 3. Rollback (Annuler)

```bash
# Annuler la dernière migration
scripts\db-migrate.bat downgrade

# Annuler les 2 dernières migrations
alembic downgrade -2

# Revenir à une révision spécifique
alembic downgrade <revision_id>

# Tout annuler (base vide)
alembic downgrade base
```

### 4. Voir l'Historique

```bash
# Historique complet
scripts\db-migrate.bat history

# Ou directement
alembic history --verbose
```

### 5. Voir la Révision Actuelle

```bash
scripts\db-migrate.bat current

# Ou directement
alembic current
```

### 6. Reset Complet

```bash
# Annuler toutes les migrations puis réappliquer
scripts\db-migrate.bat reset
```

## 🔄 Workflow Complet

### Scénario 1 : Ajouter une Colonne

#### Étape 1 : Modifier le Model

```python
# app/models/user.py
class User(SQLModel, table=True):
    id: int = Field(primary_key=True)
    email: str = Field(unique=True, index=True)
    full_name: str = Field(default="")  # ← Nouvelle colonne
    hashed_password: str
```

#### Étape 2 : Générer la Migration

```bash
alembic revision --autogenerate -m "add full_name to user"
```

**Sortie** :
```
INFO  [alembic.runtime.migration] Context impl PostgresqlImpl.
INFO  [alembic.runtime.migration] Will assume transactional DDL.
INFO  [alembic.autogenerate.compare] Detected added column 'user.full_name'
  Generating alembic\versions\20241113_1430_abc123_add_full_name_to_user.py ...  done
```

#### Étape 3 : Vérifier le Fichier Généré

```python
# alembic/versions/20241113_1430_abc123_add_full_name_to_user.py

def upgrade() -> None:
    # ### commands auto generated by Alembic - please adjust! ###
    op.add_column('user', sa.Column('full_name', sa.String(), nullable=False))
    # ### end Alembic commands ###


def downgrade() -> None:
    # ### commands auto generated by Alembic - please adjust! ###
    op.drop_column('user', 'full_name')
    # ### end Alembic commands ###
```

**⚠️ Vérifications** :
- ✅ La colonne est bien `full_name`
- ✅ Le type est correct (`String`)
- ✅ `nullable=False` est correct (on a `default=""`)
- ✅ Le downgrade supprime bien la colonne

#### Étape 4 : Appliquer la Migration

```bash
alembic upgrade head
```

**Sortie** :
```
INFO  [alembic.runtime.migration] Running upgrade  -> abc123, add full_name to user
```

#### Étape 5 : Vérifier dans PostgreSQL

```sql
-- Se connecter à la DB
psql -U aiuser -d ai_saas -h localhost -p 5435

-- Vérifier la structure
\d user

-- Résultat attendu :
--   Column      |  Type   | Nullable | Default
-- --------------+---------+----------+---------
--  id           | integer | not null |
--  email        | varchar | not null |
--  full_name    | varchar | not null |  ← Nouvelle colonne
--  hashed_password | varchar | not null |
```

#### Étape 6 : Tester le Rollback (Optionnel)

```bash
# Annuler la migration
alembic downgrade -1

# Vérifier que la colonne a disparu
psql -U aiuser -d ai_saas -c "\d user"

# Réappliquer
alembic upgrade head
```

### Scénario 2 : Ajouter une Table

#### Étape 1 : Créer le Model

```python
# app/models/subscription.py
from sqlmodel import SQLModel, Field
from datetime import datetime
from uuid import UUID, uuid4

class Subscription(SQLModel, table=True):
    __tablename__ = "subscriptions"
    
    id: UUID = Field(default_factory=uuid4, primary_key=True)
    user_id: int = Field(foreign_key="user.id")
    plan: str = Field(max_length=50)
    status: str = Field(max_length=20)
    created_at: datetime = Field(default_factory=datetime.utcnow)
```

#### Étape 2 : Importer dans env.py

```python
# alembic/env.py
from app.models.user import User
from app.models.transcription import Transcription
from app.models.subscription import Subscription  # ← Ajouter
```

#### Étape 3 : Générer la Migration

```bash
alembic revision --autogenerate -m "add subscription table"
```

#### Étape 4 : Vérifier et Appliquer

```bash
# Vérifier le fichier généré
cat alembic/versions/XXXXXX_add_subscription_table.py

# Appliquer
alembic upgrade head
```


## 🆘 Troubleshooting

### Problème 1 : "Target database is not up to date"

**Erreur** :
```
ERROR [alembic.util.messaging] Target database is not up to date.
```

**Solution** :
```bash
# Appliquer toutes les migrations en attente
alembic upgrade head
```

### Problème 2 : "Can't locate revision identified by 'XXXXX'"

**Erreur** :
```
ERROR [alembic.util.messaging] Can't locate revision identified by 'abc123'
```

**Solution** :
```bash
# Vérifier l'historique
alembic history

# Reset si nécessaire
alembic downgrade base
alembic upgrade head
```

### Problème 3 : Migration Bloquée

**Symptôme** : La migration ne se termine jamais

**Solution** :
```bash
# 1. Annuler (Ctrl+C)

# 2. Vérifier les locks PostgreSQL
psql -U aiuser -d ai_saas -c "SELECT * FROM pg_locks WHERE NOT granted;"

# 3. Tuer les connexions si nécessaire
psql -U aiuser -d ai_saas -c "SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE datname = 'ai_saas';"

# 4. Réessayer
alembic upgrade head
```

### Problème 4 : "No changes in schema detected"

**Symptôme** : Alembic ne détecte pas vos changements

**Solution** :
```bash
# 1. Vérifier que le model est importé dans env.py
cat alembic/env.py | grep "from app.models"

# 2. Vérifier que le model a `table=True`
# app/models/mymodel.py
class MyModel(SQLModel, table=True):  # ← table=True obligatoire
    ...

# 3. Régénérer
alembic revision --autogenerate -m "my changes"
```

## 📚 Exemples Complets

### Exemple 1 : Projet from Scratch

```bash
# 1. Initialiser Alembic (déjà fait dans notre projet)
alembic init alembic

# 2. Configurer env.py et alembic.ini (déjà fait)

# 3. Créer la migration initiale
alembic revision --autogenerate -m "initial tables"

# 4. Vérifier le fichier généré
```

### Exemple 2 : Évolution du Schema

```bash
# 1. Modifier models/user.py (ajouter colonne)
# 2. Générer migration
alembic revision --autogenerate -m "add bio to user"

# 3. Vérifier
cat alembic/versions/XXXXXX_add_bio_to_user.py

# 4. Appliquer
alembic upgrade head

# 5. Tester rollback
alembic downgrade -1

# 6. Réappliquer
alembic upgrade head
```

### Exemple 3 : Production Deployment

```bash
# 1. Sur dev : créer et tester la migration
alembic revision --autogenerate -m "add feature X"
alembic upgrade head
pytest

# 2. Commit et push
git add alembic/versions/
git commit -m "feat: add feature X migration"
git push

# 3. Sur staging : appliquer
git pull
alembic upgrade head
# Tester l'application

# 4. Sur prod : appliquer avec backup
# Backup DB first!
pg_dump ai_saas > backup_$(date +%Y%m%d).sql

# Appliquer migration
alembic upgrade head

# Vérifier
alembic current
```
